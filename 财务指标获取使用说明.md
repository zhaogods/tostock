# 财务指标获取使用说明

## 实施方案

**方案B：独立脚本 + 股票级缓存**

由于Tushare `fina_indicator`接口要求必须提供`ts_code`参数，无法批量获取全市场数据，采用独立作业逐股票获取并缓存的方案。

## 已完成功能

### 1. 数据字段（10个）

#### 财务指标（8个）
- `basic_eps` - 每股收益
- `bvps` - 每股净资产
- `per_capital_reserve` - 每股公积金
- `per_unassign_profit` - 每股未分配利润
- `roe_weight` - 加权净资产收益率
- `sale_gpr` - 毛利率
- `debt_asset_ratio` - 资产负债率
- `toi_yoy_ratio` - 营业收入同比增长

#### 基础信息（2个）
- `industry` - 所处行业
- `listing_date` - 上市日期

### 2. 核心文件

**新增作业**：`instock/job/fina_indicator_job.py`
- 逐股票调用`fina_indicator`接口
- 生成缓存：`instock/cache/fina/{year}/fina_{period}_all.pkl`
- 支持单日期/日期范围参数

**修改文件**：
- `instock/core/tushare_provider.py` - 简化为只读缓存
- `instock/lib/config.py` - 添加`fina_indicator`限流配置

## 使用方法

### 首次运行

```bash
# 在Docker容器内或项目环境中执行
cd /data/tostock
PYTHONPATH=/data/tostock python instock/job/fina_indicator_job.py 2024-01-15
```

**运行时间**：约34.5分钟（5525只股票 ÷ 160次/分钟）

**缓存位置**：`instock/cache/fina/2023/fina_20231231_all.pkl`

### 季度更新

财务数据按季度更新，每季度运行一次：

```bash
# Q1结束后（4月+）
python instock/job/fina_indicator_job.py 2024-04-01

# Q2结束后（7月+）
python instock/job/fina_indicator_job.py 2024-07-01

# Q3结束后（10月+）
python instock/job/fina_indicator_job.py 2024-10-01

# Q4结束后（次年1月+）
python instock/job/fina_indicator_job.py 2025-01-01
```

### 集成到主流程

缓存生成后，`fetch_stock_spot`自动使用：

```bash
# 正常运行日常作业
python instock/job/execute_daily_job.py
```

无需修改现有流程。

## 注意事项

1. **首次运行时间长**：约35分钟，建议非交易时段运行
2. **季度更新频率**：每季度运行一次即可
3. **缓存目录持久化**：Docker已配置volume映射
4. **降级兼容**：缓存不存在时，财务字段填充0（保持原有行为）
