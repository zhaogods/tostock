# tostock 数据源清单

## 概述

本项目从 InStock 原始的东方财富 `push2` 接口迁移到 **Tushare**（个股行情+资金流向）和 **东方财富/AkShare**（ETF+事件数据）的混合架构。

---

## 一、Tushare 数据源

### ✅ 1.1 完全获取的数据

#### 1.1.1 个股历史行情（stock_hist）
- **接口**: `tushare.pro.daily()`
- **字段**: 日期、开盘、收盘、最高、最低、成交量、成交额、振幅、涨跌幅、涨跌额
- **状态**: ✅ 完全获取
- **使用位置**: `tushare_provider.py:290-326` (fetch_stock_hist)
- **说明**: 用于历史K线缓存，单股票查询

#### 1.1.2 股票基本信息（stock_basic）
- **接口**: `tushare.pro.stock_basic()`
- **字段**: ts_code, name
- **状态**: ✅ 完全获取
- **使用位置**: `tushare_provider.py:115-132` (_get_stock_names)
- **说明**: 用于获取股票代码和名称映射

---

### ⚠️ 1.2 部分字段缺失的数据

#### 1.2.1 个股实时行情（stock_spot）
- **接口**: `tushare.pro.daily()` + `tushare.pro.daily_basic()`
- **完全获取字段**（来自 daily）:
  - 基础行情: date, code, close, pct_chg, change, vol, amount, open, high, low, pre_close
  
- **完全获取字段**（来自 daily_basic，当接口成功时）:
  - 估值指标: pe_ttm, pe, pb
  - 股本市值: total_share, float_share, total_mv, circ_mv
  - 换手率: turnover_rate
  - 量比: volume_ratio

- **缺失字段**（硬编码为 0 或空值）:
  ```python
  speed_increase = 0.0          # 涨速
  speed_increase_5 = 0.0        # 5分钟涨跌
  speed_increase_60 = 0.0       # 60日涨跌幅
  speed_increase_all = 0.0      # 年初至今涨跌幅
  dtsyl = 0.0                   # 动态市盈率（已有pe_ttm）
  basic_eps = 0.0               # 每股收益
  bvps = 0.0                    # 每股净资产
  per_capital_reserve = 0.0     # 每股公积金
  per_unassign_profit = 0.0     # 每股未分配利润
  roe_weight = 0.0              # 加权净资产收益率
  sale_gpr = 0.0                # 毛利率
  debt_asset_ratio = 0.0        # 资产负债率
  total_operate_income = 0      # 营业收入
  toi_yoy_ratio = 0.0           # 营业收入同比增长
  parent_netprofit = 0          # 归属净利润
  netprofit_yoy_ratio = 0.0     # 归属净利润同比增长
  report_date = None            # 报告期
  industry = ''                 # 所处行业
  listing_date = None           # 上市时间
  ```

- **状态**: ⚠️ 核心行情字段完整，财务和分时字段缺失
- **使用位置**: `tushare_provider.py:134-234` (fetch_stock_spot)
- **降级策略**: 当 daily_basic 失败时，仍使用 daily 数据并标记为 'partial_basic_missing'
- **完全降级**: 当 Tushare API/网络错误时，降级到东方财富接口

#### 1.2.2 个股资金流向（stock_fund_flow）
- **接口**: `tushare.pro.moneyflow()`
- **完全获取字段**:
  - net_mf_amount: 主力净流入金额
  - buy_elg_amount, sell_elg_amount: 超大单买卖金额
  - buy_lg_amount, sell_lg_amount: 大单买卖金额
  - buy_md_amount, sell_md_amount: 中单买卖金额
  - buy_sm_amount, sell_sm_amount: 小单买卖金额

- **缺失字段**:
  ```python
  new_price = 0.0               # 最新价
  今日涨跌幅 = 0.0               # 涨跌幅
  今日超大单净流入-净占比 = 0.0   # 超大单净占比
  今日大单净流入-净占比 = 0.0     # 大单净占比
  今日中单净流入-净占比 = 0.0     # 中单净占比
  今日小单净流入-净占比 = 0.0     # 小单净占比
  ```
  注：主力净占比通过计算得出，但各单类净占比未计算

- **状态**: ⚠️ 资金流向金额完整，净占比部分缺失
- **使用位置**: `tushare_provider.py:236-288` (fetch_stock_fund_flow)
- **限制**: 仅支持 indicator='今日'，不支持 3日/5日/10日
- **降级策略**: 失败时降级到东方财富接口

## 三、数据获取流程与降级策略

### 3.1 核心获取逻辑（stockfetch.py）

`instock/core/stockfetch.py` 是数据获取的统一门面，实现了以下降级策略：

#### 3.1.1 股票实时行情降级流程
```
fetch_stocks(date):
  1. 尝试 Tushare: ts_provider.fetch_stock_spot(date)
  2. 如果成功或部分成功 → 返回数据
  3. 如果 API_ERROR/NETWORK_ERROR:
     - 检查熔断器状态
     - 未熔断 → 降级到 stock_zh_a_spot_em()
     - 已熔断 → 返回 None
  4. 其他失败 → 返回 None
```
**代码位置**: `stockfetch.py:105-157`

#### 3.1.2 资金流向降级流程
```
fetch_stocks_fund_flow(index, date):
  1. 如果是"今日"且 Tushare 可用:
     - 尝试 ts_provider.fetch_stock_fund_flow()
     - 失败 → 降级到 stock_individual_fund_flow_rank()
  2. 其他周期（3日/5日/10日）:
     - 直接使用 stock_individual_fund_flow_rank()
```
**代码位置**: `stockfetch.py:177-205`

#### 3.1.3 历史K线降级流程
```
stock_hist_cache(code, date_start, date_end, is_cache, adjust):
  1. 检查缓存文件是否存在
  2. 如果缓存存在 → 直接返回
  3. 缓存不存在:
     - 尝试 Tushare: ts_provider.fetch_stock_hist()
     - 如果 should_retry (临时失败):
       → 降级到 stock_zh_a_hist()
     - 如果成功 → 写入缓存
  4. 返回数据
```
**代码位置**: `stockfetch.py:451-516`

---

### 3.2 限流与熔断机制

#### 3.2.1 Tushare 限流（rate_limiter.py）
- **全局限流器**: 跨线程/进程共享
- **配置方式**: 环境变量
  ```
  TUSHARE_DAILY_RATE=200        # daily接口每分钟200次
  TUSHARE_DAILY_BASIC_RATE=200  # daily_basic接口每分钟200次
  TUSHARE_MONEYFLOW_RATE=200    # moneyflow接口每分钟200次
  TUSHARE_STOCK_BASIC_RATE=200  # stock_basic接口每分钟200次
  ```
- **重试机制**: 检测频率超限错误，自动重试
- **代码位置**: `tushare_provider.py:18-23`, `tushare_provider.py:74-97`

#### 3.2.2 东方财富熔断器（eastmoney_fetcher.py）
- **触发条件**: 连续失败达到阈值
- **保护机制**: 熔断后跳过所有东方财富请求
- **状态检查**: `is_circuit_broken()`
- **影响范围**: 所有 crawling 模块
- **代码位置**: `instock/core/eastmoney_fetcher.py`

## 五、使用 Tushare 的要求

### 5.1 积分要求
- **最低积分**: 2000+ 积分
- **原因**: `daily`, `daily_basic`, `moneyflow` 接口需要较高权限
- **获取方式**: Tushare 官网注册并获取积分

### 5.2 Token 配置
```bash
# .env 文件
TUSHARE_TOKEN=your_token_here

# 或在 instock/config/tushare.json 中配置
{
  "token": "your_token_here"
}
```

### 5.3 失败处理
- Token 未配置或无效 → 启动时抛出 RuntimeError
- 积分不足 → API 调用失败，自动降级到东方财富
- 网络错误 → 自动降级到东方财富

## 八、优化建议

### 8.1 Tushare 字段补全建议

#### 高优先级（影响数据完整性）
1. **分时涨跌数据**:
   - speed_increase_5 (5分钟涨跌)
   - speed_increase_60 (60日涨跌幅)
   - speed_increase_all (年初至今涨跌幅)
   - **方案**: 使用 Tushare 历史数据计算，或保持东方财富降级

2. **财务基本面数据**:
   - basic_eps, bvps, per_capital_reserve, per_unassign_profit
   - roe_weight, sale_gpr, debt_asset_ratio
   - total_operate_income, parent_netprofit 及其同比
   - **方案**: 调用 Tushare `fina_indicator` 接口（需要2000+积分）

#### 中优先级（可选补充）
3. **行业和上市日期**:
   - industry, listing_date
   - **方案**: 从 `stock_basic` 扩展字段获取

4. **资金流向净占比**:
   - 各单类净流入占比
   - **方案**: 基于现有金额数据计算

### 8.2 架构优化建议

1. **缓存层优化**:
   - 当前: 仅历史K线有缓存
   - 建议: 为 stock_basic、交易日历等静态数据增加缓存

2. **降级策略**:

## 九、常见问题

### Q1: Tushare 数据是否完整？
**A**: 核心行情字段完整（价格、成交量等），但缺少部分财务指标和分时数据。对于量化策略，核心字段已足够。

### Q2: 为什么不全部使用 Tushare？
**A**: Tushare 不提供 ETF、选股器、龙虎榜、大宗交易等数据，且部分接口需要更高积分。

### Q3: 东方财富数据稳定吗？
**A**: 东方财富有反爬机制，项目通过代理池、Cookie、熔断器等机制提高稳定性。

### Q4: 如果 Tushare 积分不足怎么办？
**A**: 系统会自动降级到东方财富接口，不影响核心功能。

### Q5: 历史数据回填如何处理？
**A**: 使用 `backfill_job.py` 或 `execute_daily_job.py` 的日期参数进行批量回填。

## 十、总结

### 数据源分工
- **Tushare**: 个股行情（实时+历史） + 今日资金流向
- **东方财富**: ETF + 选股器 + 事件数据（龙虎榜/大宗交易等） + 降级备份
- **AkShare**: ETF行情封装 + 分红配送
- **新浪**: 交易日历（历史遗留）

### 数据完整性
- **完全获取**: 股票历史K线、股票基本信息
- **主要字段获取**: 股票实时行情（60%）、资金流向（70%）
- **缺失字段**: 主要是财务指标和分时数据，对核心量化功能影响有限

### 可靠性保障
- Tushare 失败 → 自动降级东方财富
- 东方财富有熔断器保护
- 历史数据有磁盘缓存
